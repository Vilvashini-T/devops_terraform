# 🏗️ Infrastructure Architecture

## Overview

This project deploys a full-stack MERN web application on AWS using Terraform. The entire infrastructure — server, firewall, static IP, SSH keys, and app setup — is created automatically by running a single command.

## AWS Resources Created by Terraform

| Resource | Name in Code | What it does |
|---|---|---|
| EC2 Instance | `app_server` | The virtual Linux server that runs our entire app |
| Elastic IP | `app_eip` | A permanent static public IP — does not change on restart |
| Security Group | `app_sg` | Cloud firewall — only opens the ports our app needs |
| IAM Role | `ssm_role` | Gives the server permission to use AWS Systems Manager |
| SSH Key Pair | `aws_key_pair.app_key` | Registers the public key with AWS for server login |
| TLS Private Key | `tls_private_key.app_key` | Auto-generates the RSA 4096-bit private key locally |
| Local File | `local_file.private_key` | Saves the `.pem` private key file to your machine |

## What Happens When the Server Boots (Automatic Setup)

When Terraform creates the EC2 instance, it passes a `user_data` bash script that runs automatically on the blank server. Here is what it does, step by step:

```
Step  1 → Update Linux packages
Step  2 → Install Node.js version 20
Step  3 → Install MongoDB version 7.0
Step  4 → Wait until MongoDB is ready (checks every 2 seconds, up to 60 seconds)
Step  5 → Install PM2 (process manager) and serve (static file server) globally
Step  6 → Clone the MERN app from GitHub into /home/ubuntu/app
Step  7 → Create backend .env file (sets MONGO_URI and PORT=5000)
Step  8 → Install backend npm dependencies
Step  9 → Seed the database with initial data (node seeder.js)
Step 10 → Start the backend server with PM2
Step 11 → Detect the server's own public IP address automatically
Step 12 → Create frontend .env file (sets VITE_API_BASE_URL to the detected IP)
Step 13 → Build the React frontend for production (npm run build)
Step 14 → Serve the frontend on port 80 using PM2
Step 15 → Save PM2 process list so apps restart automatically after a reboot
```

## Security Design Decisions

| Decision | Why |
|---|---|
| SSH keys are auto-generated by Terraform | No need to manually create keys in AWS Console. Zero human error. |
| `.pem` file and `.tfstate` are in `.gitignore` | These files contain secrets. They must NEVER be pushed to GitHub. |
| IAM Role uses least privilege | The server only has SSM access — no other AWS permissions. |
| Security Group blocks all ports by default | Only ports 22, 80, 3000, and 5000 are explicitly opened. |
